Retrieves the hash user identifier currently in use. If no user identifer has been configured, Purple Robot will use the device's default Google account. This hash is used to identify the source of sensor data and other probe readings uploaded to a remote Purple Robot endpoint.

The user identifier may be hashed using either the MD5 or SHA-2 hashing algorithms. MD5 is the default algorithm for legacy compatibility, but SHA-2 may be selected in the Purple Robot settings, under the Preferred Hash Function option on the main settings screen.

To configure the selected algorithm, simply set the configuration value of config_preferred_hash_function to the preferred algorithm: 

PurpleRobot.updateConfig("config_preferred_hash_function", "SHA-2");

// or...

PurpleRobot.updateConfig("config_preferred_hash_function", "MD5");

If the generated hash contains less than 32 characters, the hash will be zero-padded until it reaches a 32-character length.

Arguments

None

Return Values

String containing the hash of the active user identifier.

JavaScript Example

PurpleRobot.fetchUserHash();

Scheme Example

(pr-fetch-user-hash)
