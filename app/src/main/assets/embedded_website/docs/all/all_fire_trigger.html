Overrides any tests and fires the trigger identified by triggerId.

Note that if this function is called for a date-time trigger using the detetime_random parameter and the trigger is in an active period and has not yet fired, firing the trigger manually will prevent the trigger for firing again during the current active period automatically. See PurpleRobot.updateTrigger for more details.

Arguments

identifier - Unique identifier of the trigger to fire.

Return Values

None

JavaScript Example

var dateTrigger = {
    "identifier": "hourly-chime",
    "name": "Hourly Chime",
    "type": "datetime",
    "action": "PurpleRobot.playDefaultTone();",
    "datetime_start": "20150327T130000",
    "datetime_end": "20150327T130100",
    "datetime_random": false,
    "datetime_repeat": "FREQ=HOURLY;INTERVAL=1;",
    "fire_on_boot": true
};

PurpleRobot.updateTrigger('hourly-chime', dateTrigger);

PurpleRobot.fireTrigger("hourly-chime");

Scheme Example

(pr-update-trigger '(("identifier" . "hourly-chime")
                     ("name" . "Hourly Chime")
                     ("type" . "datetime")
                     ("action" . "(pr-play-default-tone)")
                     ("datetime_start" . "20150327T130000")
                     ("datetime_end" . "20150327T130100")
                     ("datetime_random" . #f)
                     ("datetime_repeat" . "FREQ=HOURLY;INTERVAL=1;")
                     ("fire_on_boot" . #t)))

(pr-fire-trigger "hourly-chime")
